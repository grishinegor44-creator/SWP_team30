# Sprint 1 Retrospective

**Date:** 21.06.2026
**Participants:** @Son-Go, @the-shtorm, @venimu, @Zhend0sss, @grishinegor44-creator

## What went well

1. **Role splitting:** team naturally splitted on frontend, backend and devops part, where each team member completed his own tasks
2. **Sprint planning:** team planned exactly optimal amount of necessary work for sprint (all necessary tasks were completed, 0 optional were touched)
3. **Code review:** team efficiently reviewed peers work within hours, so no review bottleneck appeared

## What did not go well

1. **Naming workflow** due to exceptional ammount of commits, team failed to implement `<issue-number>-description` commit naming and wide PR descriptions
2. **Deployment process:** Project containers several times went down due to lack of DevOps knowlegde
3. **Misscomunication:** Meetings without all team members generated several knowledge gaps, which lead to several bugs and API misscomunications

## Action points

1. Team will search to container testing tool to improve overall deploy stability
2. All meetings will be documented and notes will be published on Miro board
